안정적 학습을 위한 단일 최선의 시뮬레이션 안

문서 목적

이 문서는 현재 문제를 다시 압축한 뒤, 실제로 가장 먼저 구현해야 할
단 하나의 최선안을 정의한다.

핵심 전제는 다음과 같다.

- 256차원 feature embedding은 학습 안정성을 반영하는 표현 공간이다.
- 오른쪽 학습 버퍼의 분포가 왼쪽 Mother dataset 분포를 잘 추종할수록
  학습이 안정적일 가능성이 높다.
- 운영 환경에서는 버퍼 크기 N, 최대 보관 기간 K일, FIFO 성격을 반드시
  만족해야 한다.

따라서 이 문제의 본질은 단순한 novelty filtering이 아니다.
진짜 문제는 다음과 같다.

"시간 순서대로 들어오는 샘플 스트림 아래에서, 크기 N과 기간 K의 FIFO 버퍼가
Mother distribution을 가장 안정적으로 복제하도록 유지 정책을 설계하는 것"


최종 결론

가장 먼저 구현해야 할 단일 최선안은 다음이다.

"고정 참조공간 기반 Mother-Quota FIFO 버퍼 + 지역 중복 억제 거리 규칙"

더 짧게 말하면,
전역적으로는 Mother 분포 비율을 맞추고,
로컬하게는 너무 가까운 중복 샘플만 억제하는 방식이다.

즉, 정책의 중심은 quota이고,
거리 threshold는 보조 규칙이다.

이 구성이 최선인 이유는 다음과 같다.

- Mother 분포 추종이라는 전역 목표를 직접 정책에 넣을 수 있다.
- N, K, FIFO를 구조적으로 강제할 수 있다.
- 거리 기반 규칙이 잘하는 일인 로컬 중복 억제를 보조적으로 살릴 수 있다.
- 순수 거리 threshold 방식의 핵심 결함인 density 왜곡을 피할 수 있다.
- 시뮬레이션 결과를 해석하고 디버깅하기 쉽다.


왜 순수 거리 threshold 방식만으로는 부족한가

직관적으로는 "가까운 점이 이미 있으면 안 넣고, 없으면 넣는다"가
적당한 밀도를 만들 것처럼 보인다. 하지만 이 방식 단독으로는
Mother 분포 추종을 보장하지 못한다.

이유는 간단하다.

- 거리 threshold 규칙은 로컬 규칙이다.
- Mother 분포 추종은 전역 목표다.

거리 규칙이 직접 보는 것은:
- 새 샘플이 기존 버퍼와 얼마나 가까운가

하지만 우리가 맞추고 싶은 것은:
- Mother에서 각 영역이 차지하는 비율
- 현재 버퍼가 그 비율을 얼마나 잘 재현하는가
- 어떤 샘플을 빼야 전체 분포 오차가 줄어드는가

즉 거리 threshold만으로는 다음이 보장되지 않는다.

- 밀집 영역이 원래 많이 차지해야 하는 비율
- 희소 영역이 원래 적게 차지해야 하는 비율
- 시간 흐름 속에서의 안정적 분포 유지
- FIFO와 K 제약을 걸었을 때의 분포 보존

정리하면,
순수 거리 방식은 support coverage를 어느 정도 만들 수는 있어도,
Mother density ratio를 유지하는 정책은 아니다.

따라서 거리 규칙은 메인 정책이 아니라,
이미 정해진 전역 목표를 보조하는 로컬 regularizer로 쓰는 것이 맞다.


왜 이 최선안이 목적에 가장 맞는가

이 최선안은 문제를 두 층으로 나눈다.

1. 전역 층: Mother 분포를 얼마나 닮게 유지할 것인가
2. 로컬 층: 같은 영역 안에서 너무 비슷한 샘플이 중복되지 않게 할 것인가

전역 층은 quota가 담당한다.
로컬 층은 거리 규칙이 담당한다.
시간 제약은 FIFO와 K일 만료가 담당한다.

즉 역할이 명확하게 분리된다.

- quota: 분포 비율 보존
- threshold: 중복 억제
- FIFO: 운영 일관성
- K일 만료: 신선도 보장

이 구조가 중요한 이유는,
학습 안정성 목표를 흔드는 가장 큰 원인이 보통 두 가지이기 때문이다.

- 특정 영역의 과대표집 또는 과소대표집
- 최근에 비슷한 샘플이 과도하게 유입되어 버퍼가 국소적으로 뭉치는 현상

quota는 첫 번째 문제를 잡고,
threshold는 두 번째 문제를 잡는다.


정책 정의

1. 고정 참조공간 정의

정책은 시각화용 UMAP 2D/3D를 직접 기준으로 삼지 않는 것이 좋다.
UMAP은 시각화에는 좋지만, 정책 기준 공간으로 쓰기에는 변동성과 왜곡이 있다.

정책 기준은 다음처럼 두는 것이 적절하다.

- Mother 256D feature에 대해 표준화 수행
- Mother 전체에 고정 PCA를 fit
- 운영용 참조공간은 PCA 8~16차원 정도로 유지

이유는 다음과 같다.

- 2D/3D보다 정보 손실이 적다.
- 고정 변환이므로 신규 샘플에도 일관되게 적용할 수 있다.
- quota와 거리 계산 모두 안정적으로 수행할 수 있다.

시각화는 기존처럼 PCA/UMAP 2D/3D로 보여주되,
정책 판단은 고정 PCA 참조공간에서 수행하는 것이 맞다.


2. Mother 지역 분할과 목표 quota 정의

참조공간 위에서 Mother dataset을 고정된 영역들로 나눈다.
방법은 다음 중 하나가 적합하다.

- K-means cluster
- coarse grid binning
- density-aware region split

가장 먼저는 K-means가 실용적이다.

각 영역 c에 대해 Mother 비율 p_c를 계산하고,
버퍼 크기 N에 대해 목표 개수 q_c를 정한다.

기본식은 다음과 같다.

- p_c = Mother 내 영역 c 비율
- q_c = round(N * p_c)

필요하면 희귀하지만 중요한 영역에는 최소 quota를 둘 수 있다.

이 단계가 핵심이다.
이제 정책은 "가까운지"만 보지 않고,
"현재 어떤 영역이 부족한지"를 직접 볼 수 있게 된다.


3. 버퍼 상태

각 샘플은 최소한 아래 정보를 가진다.

- feature vector
- timestamp
- 참조공간 좌표
- 소속 영역 c
- 최근접 거리 또는 local redundancy score

버퍼는 아래 상태를 유지한다.

- 전체 active sample 목록
- 영역별 현재 개수 n_c
- timestamp 기준 FIFO 순서
- K일 만료 대상


4. 새 샘플 처리 순서

새 샘플 x_t가 시간 t에 들어오면 아래 순서로 처리한다.

1. 먼저 K일을 초과한 샘플을 모두 제거한다.
2. x_t를 고정 PCA 참조공간으로 투영한다.
3. x_t가 속한 영역 c를 찾는다.
4. 해당 영역 또는 인접 영역에서 최근접 거리를 계산한다.
5. quota 상태와 거리 상태를 함께 보고 accept 여부를 판단한다.
6. accept 후 크기가 N을 초과하면 eviction을 수행한다.

핵심은 accept와 eviction이 분리되면 안 된다는 점이다.
새 샘플을 넣는 판단과 기존 샘플을 빼는 판단은 항상 한 세트여야 한다.


5. accept 규칙

가장 단순하고 강건한 규칙은 아래와 같다.

- 영역 c가 quota 미달이면 우선 채택 후보로 본다.
- 단, 거의 동일한 중복 샘플이면 reject할 수 있다.
- 영역 c가 quota 초과 또는 quota 충족 상태이면,
  새 샘플이 기존 샘플보다 더 좋은 대표성 또는 더 높은 신선도를 줄 때만 채택한다.

여기서 거리 threshold는 이렇게 쓰는 것이 맞다.

- 같은 영역 안에서 최근접 거리가 너무 작으면 redundancy가 높다고 본다.
- 단, quota가 크게 부족한 영역이라면 threshold만으로 무조건 막지 않는다.

즉 거리 규칙은 quota보다 상위 규칙이 아니다.
분포 복원 필요가 더 우선이다.


6. eviction 규칙

이 최선안의 핵심은 사실 accept보다 eviction이다.
버퍼가 N을 초과했을 때 무엇을 빼느냐가 분포를 결정한다.

제거 순서는 아래가 가장 합리적이다.

- 1순위: K일 초과 샘플
- 2순위: quota를 초과한 영역의 샘플
- 3순위: 그중에서도 가장 오래되고 가장 중복적인 샘플
- 4순위: quota를 깨지 않는 범위에서 FIFO 우선 제거

즉 원칙은 이렇다.

- 분포를 해치지 않는다면 오래된 것을 먼저 뺀다.
- 분포를 크게 해친다면 단순 FIFO만으로 제거하지 않는다.

이 규칙이 있어야 FIFO와 Mother 추종이 동시에 성립한다.


7. priority queue를 어디에 쓰는가

priority queue는 단독 알고리즘이 아니라,
이 정책을 효율적으로 구현하는 자료구조로 쓰는 것이 맞다.

구체적으로는 다음처럼 쓸 수 있다.

- 만료용 min-heap: timestamp 기준
- 영역별 FIFO queue: 각 영역 내부의 오래된 순서 관리
- eviction 후보 heap: quota 초과량, age, redundancy를 반영한 제거 우선순위

즉 priority queue 자체가 정책은 아니다.
정책은 quota-first이고,
priority queue는 그 정책을 빠르게 실행하게 해 주는 구현 수단이다.


추천 점수식

가장 먼저 구현할 때는 복잡한 최적화보다,
아래처럼 단순한 점수식이 적절하다.

EvictPriority(i) =
  a * OverQuota(region_i)
+ b * Redundancy(i)
+ c * Age(i)

여기서 의미는 다음과 같다.

- OverQuota(region_i): 해당 영역이 quota를 얼마나 초과했는가
- Redundancy(i): 같은 영역 내에서 얼마나 대체 가능한가
- Age(i): 얼마나 오래된 샘플인가

점수가 높을수록 먼저 제거한다.

AcceptScore(x_new)는 더 단순하게 두는 것이 좋다.

- quota deficit가 크면 우선 채택
- 지역 중복이 지나치게 크면 감점
- 최신성 이점이 있으면 가점

처음 버전은 복잡한 score보다 rule-based로 시작하고,
필요할 때만 score weight를 학습 또는 튜닝하는 것이 맞다.


정책의 목적 함수

이 정책을 한 줄의 수식 감각으로 쓰면 다음과 같다.

목표:
- Mother와 버퍼의 영역별 분포 오차를 최소화
- 버퍼 내 과도한 로컬 중복을 억제
- 평균 age를 낮게 유지

제약:
- |Buffer| <= N
- age <= K days
- FIFO 성격 유지

즉 최적화 방향은 다음과 같다.

Minimize:
DistributionMismatch(Buffer, Mother)
+ lambda_dup * Redundancy(Buffer)
+ lambda_age * Staleness(Buffer)

subject to:
- size <= N
- max age <= K
- FIFO-compatible eviction

이 formulation이 지금 목적과 가장 정합적이다.


실제 구현 순서

최초 구현은 아래 단계로 가는 것이 가장 좋다.

1. Mother 전체에 고정 PCA fit
2. PCA 참조공간에서 K-means cluster 생성
3. cluster별 Mother quota 계산
4. K일 만료 + N 제한 + FIFO 버퍼 구현
5. quota 기반 accept/evict 규칙 구현
6. 같은 cluster 안의 distance-threshold redundancy 억제 추가
7. 날짜별 분포 추종 지표를 로그 및 시각화로 출력

처음부터 복잡한 scoring system 전체를 구현할 필요는 없다.
가장 먼저는 quota-first rule-based 버전이 맞다.
그 뒤에만 threshold와 score를 세밀하게 다듬으면 된다.


반드시 확인해야 할 평가 지표

이 정책이 맞는지 보려면 acceptance rate가 아니라 아래를 봐야 한다.

- 날짜별 cluster ratio error
- Mother 대비 coverage gap
- 버퍼 평균 age
- 버퍼 내 redundancy 수준
- 날짜별 drift 발생 시 버퍼 회복 속도
- 학습 성능의 분산 또는 안정성 지표

즉 최종 평가는 "얼마나 많이 받았는가"가 아니라,
"얼마나 Mother를 안정적으로 닮게 유지했는가"이다.


최종 한 줄 결론

가장 먼저 구현해야 할 최선의 단일 안은,
"고정 PCA 참조공간에서 Mother 분포 quota를 유지하고,
각 지역 내부에서는 거리 threshold로 중복만 억제하는 Quota-First FIFO 버퍼 정책"이다.

이 안이 가장 좋은 이유는,
- 전역적으로는 Mother 추종을 직접 보장하려 하고
- 로컬하게는 거리 기반 직관을 살리며
- 운영적으로는 N, K, FIFO를 동시에 만족시키기 때문이다.
